LCS Algorithm Comparison

Analyzing Time and Space Complexity of the Three Approaches

Algorithm Time Complexity Space Complexity Best For Backtracking?
Standard DP O(N × M) O(N × M) General use, small to medium strings, learning. Easy — full table available
Rolling Array O(N × M) O(min(N, M)) Memory-constrained environments where only the length is needed. Hard — previous rows discarded
Hirschberg's O(N × M) O(min(N, M)) Large sequences (DNA) where space is critical but the LCS string is needed. Complex but possible

Summary

Standard DP is the most intuitive approach. It builds a full (N+1)×(M+1) table, making backtracking straightforward. However, it uses the most memory — O(N×M) space.

Rolling Array optimizes space by realizing we only need the previous row to compute the current row. This reduces space to O(min(N,M)) but destroys the path information, making it hard to reconstruct the actual LCS string.

Hirschberg's Algorithm combines divide-and-conquer with the rolling array technique. It achieves O(min(N,M)) space while still recovering the full LCS string — the gold standard for linear-space LCS computation.
